Kingsley BookDrop Interface

The software supports an interface to a Kingsley BookDrop hardware. The Kingsley BookDrop is a simple, mechanical item return box, which can be locked or unlocked. This function is meant as an emergency return option in cases where the application software is unable to perform its function (for example, if the LMS is unavailable or another vital hardware element such as the RFID reader is out of service). In such a case the machine controller will open a "Kingsley BookDrop" as an "emergency" return option to the patrons.

In normal operation, the application will answer the "Idle" messages sent aprox. every 10s by the machine software with the respective acknowledgement. In case the Bibliotheca-SW has a problem & cannot provide return services anymore, it will cease to answer those messages. This will inform the machine, that all items inserted into the unit must be rejected immediately and that the drop must be opened. In this scenario the machine software will continue to send the idle messages to the application; once the problem has gone, those messages will be answered again & the system will return to normal operation mode.

No configuration is required to support this option - the default as configured in "backend.io.output" must not be changed. test yes test again

backend.io.output - Contains system specific rules related to the door configuration of the installation.

Sample contents of the backend.io.output config key:

<XMLConfigurationStruct>

<Key>backend.io.output</Key>

<Value>

&lt;?xml version="1.0" encoding="utf-8"?&gt;

&lt;string&gt;

[OutOfOrder]

gilgen.drop = 0x01

[NormalOperation]

gilgen.drop = 0x00

[OpenCapellaServiceDoor]

gilgen.serviceDoor = 0x01

[CloseCapellaServiceDoor]

gilgen.serviceDoor = 0x00

&lt;/string&gt;

</Value>

</XMLConfigurationStruct>
